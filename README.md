# শিমু - ভার্চুয়াল হার্ডওয়্যার সিমুলেটর

**শিমু** হলো নির্দেশিকা-১৬ (Nirdeshika-16) যন্ত্র-বিধানের জন্য নির্মিত একটি রেফারেন্স সিমুলেটর বা ভার্চুয়াল সিপিইউ। এটি 'বয়ন' অ্যাসেম্বলার থেকে প্রাপ্ত সার্বভৌম র-বাইনারি (.ব) ফাইলগুলো সরাসরি মেমোরিতে লোড করে প্রসেসরের ন্যায় কার্যকর করতে পারে।

## প্রধান কারিগরি বৈশিষ্ট্য

- **ইনস্ট্রাকশন সাইকেল:** এটি একটি পূর্ণাঙ্গ ফেচ-ডিকোড-এক্সিকিউট (Fetch-Decode-Execute) লুপ অনুসরণ করে।
- **ভার্চুয়াল মেমোরি:** ৬৫,৫৩৬ বাইটের (৬৪ কিলোবাইট) একটি সম্পূর্ণ স্বতন্ত্র র‍্যাম সিমুলেশন।
- **সার্বভৌম আউটপুট ইঞ্জিন:** এটি গাণিতিক ফলাফল এবং রেজিস্টারের অবস্থা প্রদর্শনের জন্য নির্দেশিকা-১৬ এর নিজস্ব 'ষোড়শিক' (Akkhar-Hex) এবং বাংলা সংখ্যা পদ্ধতি ব্যবহার করে।
- **বিট ডিকোডিং:** ৪-৪-৮ নির্দেশ বিন্যাস অনুযায়ী হার্ডওয়্যার লেভেল ডিকোডিং।

## প্রোজেক্ট মডিউলসমূহ

- `src/cpu.c`: সিপিইউ-র প্রধান ইঞ্জিন ও ইনস্ট্রাকশন এক্সিকিউশন লজিক।
- `src/memory.c`: ভার্চুয়াল র‍্যাম ব্যবস্থাপনা।
- `src/utils/formatting.c`: আউটপুটকে বাংলা সংখ্যা ও হেক্স পদ্ধতিতে রূপান্তরকারী ইঞ্জিন।
- `src/main.c`: বাইনারি লোডার এবং সিমুলেটর এন্ট্রি পয়েন্ট।

## ব্যবহার নির্দেশিকা

### বিল্ড করার পদ্ধতি (Compilation)
সিমুলেটরটি কম্পাইল করতে GCC ব্যবহার করে নিচের কমান্ডটি দিন:

```bash
gcc -Iinclude src/main.c src/cpu.c src/memory.c src/alu.c src/io.c src/utils/formatting.c -o bin/shimu
```

### সিমুলেশন চালানোর পদ্ধতি
একটি অ্যাসেম্বল করা `.ব` ফাইল রান করতে:

```bash
./bin/shimu ফাইল_পাথ.ব
```
সিমুলেশন শেষ হওয়ার পর এটি স্বয়ংক্রিয়ভাবে একটি `সিমুলেটর_ফলাফল.txt` ফাইল তৈরি করবে যেখানে প্রসেসরের চূড়ান্ত অবস্থা ও আউটপুট বাংলা ফরম্যাটে সংরক্ষিত থাকবে।
